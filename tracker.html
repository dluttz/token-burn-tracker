Where you can become more efficient ?

How to change the way you work to burn fewer tokens — your heaviest chats and folders, plus a guide to which tool to reach for and how to prompt it. (Background jobs to switch off are in “suggested to remove” below.)

When you want to…UsePrompt it like
A quick fact, summary, or renameA small/fast model (Haiku, GPT-mini)“In one line: …” — keep it short, give no extra context.
Focused code edits in one repoCodexPoint it at the exact file/function and describe the change precisely.
Multi-file work, refactors, researchClaude Code / CoworkState the goal + constraints once, then let it plan and use sub-agents.
Deep reasoning or tricky designOpus — sparinglyGive full context once; ask for a plan before any code.
A session that's getting expensive/compact or a fresh threadSummarize what's done, start clean — stops re-reading old context every turn.
